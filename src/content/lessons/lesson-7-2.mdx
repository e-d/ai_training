## The Stack Behind the Magic

Building an LLM-powered application involves much more than selecting a model. Behind every production AI application is a technology stack — layers of software that handle everything from processing user input to retrieving relevant data to formatting the model's output for the end user. Understanding this stack helps business leaders make informed decisions about vendor selection, build-vs-buy trade-offs, and where their organizations need to invest.

## Model Providers: The Foundation Layer

At the base of every LLM application is the model itself. The market has consolidated around a few major providers, each with distinct strengths:

**OpenAI** remains the market leader. GPT-4 and its successors power applications from customer service to code generation. As of early 2025, OpenAI reported over 2 million developers using its API and more than 92% of Fortune 500 companies using its products. OpenAI's strength is breadth — its models perform well across a wide range of tasks without extensive customization.

**Anthropic** has built its reputation on safety and reliability. Claude, Anthropic's flagship model, is used extensively in enterprise settings where accuracy and predictable behavior matter — legal analysis, financial services, and healthcare. Anthropic's Constitutional AI approach, which trains models to follow explicit principles, appeals to organizations in regulated industries.

**Google DeepMind** offers Gemini, a multimodal model family that processes text, images, audio, and video natively. Google's advantage is integration with its cloud infrastructure — BigQuery, Vertex AI, and Google Workspace — making Gemini a natural choice for organizations already invested in the Google ecosystem.

**Meta's Llama** represents the open-source alternative. Llama models can be downloaded, modified, and hosted on your own infrastructure — eliminating data-sharing concerns and recurring API costs. Companies like Bloomberg and Goldman Sachs have invested in fine-tuning open-source models on proprietary data, creating custom models that never send sensitive information to a third-party API.

### The Multi-Model Reality

Most enterprise deployments don't rely on a single model. A 2024 Andreessen Horowitz survey found that 85% of enterprise AI teams use models from at least two providers, and 44% use models from three or more. The reasons are practical: different models excel at different tasks, pricing varies significantly, and vendor diversification reduces dependence on any single provider's uptime and pricing decisions.

## Orchestration Frameworks: The Connective Tissue

Raw model APIs are powerful but primitive. Orchestration frameworks sit between your application and the model, handling the complex logic that production systems require: managing prompts, chaining multiple model calls, connecting to external data sources, and handling errors.

**LangChain** is the most widely adopted orchestration framework, with over 90,000 GitHub stars and a rapidly growing ecosystem. LangChain provides standardized components for common patterns — retrieval-augmented generation, agent tool use, conversational memory — so developers don't rebuild them from scratch for each project. Think of it as the middleware that makes LLM applications composable and maintainable.

**LlamaIndex** specializes in data ingestion and retrieval. While LangChain is a general-purpose orchestration tool, LlamaIndex focuses specifically on connecting LLMs to your data — ingesting documents from dozens of sources, chunking them intelligently, creating searchable indexes, and optimizing retrieval quality. Organizations with large document repositories often use LlamaIndex alongside LangChain.

**Semantic Kernel**, developed by Microsoft, integrates deeply with the Azure ecosystem and .NET development. For organizations whose infrastructure runs on Microsoft technology, Semantic Kernel provides a familiar development model with native Azure OpenAI Service integration.

**Haystack** by deepset offers an open-source framework particularly strong in search and question-answering applications. It's a popular choice in European enterprises due to its German origins and strong data privacy posture.

## Embedding Models: Turning Text Into Math

A technology that underlies much of the LLM application stack is the embedding model — a specialized model that converts text into numerical vectors (arrays of numbers) that capture semantic meaning. When two pieces of text are about the same topic, their embedding vectors are mathematically similar. This is what makes semantic search possible.

When a user asks a question and the system needs to find relevant documents, it doesn't search for keyword matches. It converts the question to a vector using an embedding model, then finds documents whose vectors are closest in the mathematical space. This is why a search for "employee termination policies" can find a document titled "Workforce Reduction Guidelines" — the meanings are similar even though the words are different.

OpenAI's text-embedding-3 family, Cohere's Embed v3, and open-source models like BGE and E5 are common choices. The selection matters: embedding quality directly determines retrieval quality, which directly determines whether your RAG application returns accurate answers or irrelevant ones.

## User Interface Frameworks: Where Users Meet the Model

The final layer is the interface that end users actually interact with. Two frameworks have emerged as dominant choices for building LLM application front-ends:

**Streamlit**, acquired by Snowflake in 2022, lets developers build interactive web applications entirely in Python. It's the fastest path from prototype to deployed application — a functional chatbot interface can be built in under 50 lines of code. Streamlit's simplicity makes it the default choice for internal tools and proof-of-concept applications.

**Gradio**, backed by Hugging Face, serves a similar niche with stronger support for machine learning demonstrations and multimodal interfaces. Gradio is the standard interface for Hugging Face's model hub, where thousands of models are available for experimentation.

For production-grade external-facing applications, organizations typically use standard web frameworks (React, Next.js, or Vue.js) with custom integrations to LLM APIs. Vercel's AI SDK has emerged as a popular choice for building streaming chat interfaces in production web applications.

## Infrastructure and Observability

Production LLM applications require infrastructure that traditional software doesn't:

- **Vector databases** (covered in depth in the next lesson) store and retrieve the embedding vectors that power semantic search
- **Prompt management platforms** like PromptLayer and Humanloop track prompt versions, measure performance across model versions, and enable A/B testing of different prompts
- **LLM observability tools** like LangSmith, Helicone, and Weights & Biases monitor token usage, latency, cost, and response quality in production — the AI equivalent of application performance monitoring
- **Guardrail frameworks** like Guardrails AI and NeMo Guardrails enforce output constraints, content policies, and safety rules on model responses before they reach the user

> **Key takeaway:** A production LLM application is a stack, not a single model. Understanding the layers — model providers, orchestration frameworks, embedding models, interfaces, and observability tools — equips business leaders to evaluate vendor proposals, estimate realistic project timelines, and ask the right questions of their technical teams.
